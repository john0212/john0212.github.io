Spring Boot IoC 重點

By John 21 November 2022

2-2 章

@component
  1. 用法:只能加在 class 上
  2. 用途:將該 class 變成由 Spring 容器所管理的 object ，那這些 object 會被稱作為 bean(這個 bean 名字為 class name 的第一個字母轉小寫)
專有名詞
  1. Ioc(Inversion of Control)控制反轉:會將 object 存放在 spring 容器裡面
  2. DI(Dependency Injection)依賴注入:會讓我們去取得存放在 spring 容器裡面的那些 object
  3. bean:存放在 spring 容器裡的 object
  4. @Autowired:加在變數上，取得 spring 容器中的 bean

2-3 章

@Autowired
  1. 用法:通常加在 class 變數上
  2. 用途:根據變數的類型，去 spring 容器中尋找有沒有符合類型的 bean
@Qualifier
  1. 用法:通常是加在 class 變數上，會跟@Autowired 一起使用
  2. 用途:指定要載入的 bean 的名字

2-4 章

Bean 的創建方式，這兩個是可以同時使用的
  1. 在 class 上加上@Component 註解
  2. 使用@Configuration+@Bean 註解
@Configuration
  1. 用法:只能加在class上
  2. 用途:spring 中的設定用註解，表示這個 class 是拿來設定 spring 用的
@Bean
  1. 用法:只能加在帶有 @Configuration class 的方法上
  2. 用途:在 spring 容器中創建一個 bean

2-5 章

spring 中初始化 bean 的方法，一次只要選一種使用就好
  1. 使用@PostConstruct(較常用)，有幾個細節要注意
    • 方法一定要為 public
    • 返回類型一定要是 void
    • 方法名字可以隨意取
    • 不能有參數
@PostConstruct
public void initialize() {
    // 要做的事
}
  1. 實現 InitializingBean interface 的 afterPropertiesSet() 方法
// 要先實作 InitializingBean，之後去實作 afterPropertiesSet() method
@Override
public void afterPropertiesSet() throws Exception {
  // 要做的事
}

2-6 章

Bean 的生命週期重點
  1. Bean 的生命週期 : 創建->初始化->可以被別人拿去使用
  2. 創建時若有依賴其他 Bean，則 Spring 會回過頭去 [創建+初始化] 那個被依賴的 Bean
  3. 不要寫出循環依賴的 code

2-7 章

Spring Boot 設定檔 - application.properties
  1. 用法:使用 properties 語法 (key=value)
  2. 用途:存放 Spring Boot 的設定值
@Value
  1. 用法:加在Bean或是設定 Spring 用的 class裡面的變數上
  2. 用途:讀取 Spring Boot 設定檔 (application.properties) 中指定的 Key 的值
  3. 使用@Value 有幾個重點要注意
    • 可以加在帶有@Component 或@Configuration 上
    • 格式必須為 ${key}
    • 變數所使用的類型，必須要跟 Spring Boot 設定檔中那個 key 所對應的類型一樣才行
    • 可以設定預設值。當找不到對應的 key 時，就會用此預設值。
